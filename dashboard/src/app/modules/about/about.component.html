Structure The application is client-server

The program running on the client is the front-end (service user), the one running on the server is the back-end (service provider). The back-end creates an API that the front-end uses. Thus, the front-end developer does not need to know the features of the deployment, and the back-end developer does not need to know the front-end deployment. The backend is implemented in java (designed as a Maven project - def: Maven is a project building and management system, written in Java. The central concept in Maven is that of POM (Project Object Model), an XML document that describes a project or a module thereof.

Architecture Documentation

Most web applications are developed according to MVC (Model-View-Controller), which is a generic architecture of a web application. The data structuring model is separated from the way of processing (application control, business logic) and the way of presenting them (Web interface) using the principle of demarcation of responsibilities (separation of concerns). Controller is responsible for receiving requests from the client (GET / POST requests issued based on user actions) and manages the resources needed to satisfy requests. Usually, it will call a model according to the requested action and then select an appropriate view.
